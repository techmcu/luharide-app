## 🔧 Technical Implementation

### **File Structure:**
```
lib/screens/home/
├── home_screen.dart              (Router based on role)
├── passenger_home_screen.dart    (Passenger UI)
├── driver_home_screen.dart       (Driver UI)
└── union_admin_home_screen.dart  (Admin UI)
```

### **Role Routing Logic:**
```dart
// In home_screen.dart
switch (role) {
  case 'driver':
    return const DriverHomeScreen();
  case 'union_admin':
    return const UnionAdminHomeScreen();
  case 'passenger':
  default:
    return const PassengerHomeScreen();
}
```

### **State Management:**
- Uses Provider for auth state
- Real-time user data
- Automatic role detection
